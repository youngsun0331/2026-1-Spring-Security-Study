# Security-1주차

## 인증과 인가
### 인증 (Authentication)
+ 로그인
+ ex) 학생증 발급

### 인가 (Authorization)
+ 리소스 접근 제어
+ ex) 학교 도서관 접근, 교수님 연구실, 홍문관 옥상은 왜 안열리냐

## Spring Security
+ Spring 환경 인증 / 인가 담당 하는 프레임 워크
    + 프레임 워크 : 툴
    + 라이브러리 : 도서관

## 서블릿 필터(Servlet Filter)
+ 요청/응답을 가로채서 전처리/후처리
+ ex) 요청 로깅 , 인코딩 설정

## Spring Security의 원리
+ 보안을 담당하는 서블릿 필터를 끼워넣자
+ 이름은 SercurityFilterChain으로 하자

## Q1
>Q1. 근데 서블릿 필터 AOP랑 똑같은거 아님?

+ 동작하는 위치/목적이다름
    + AOP는 비즈니스적  
    + 필터와 인터셉터는 인증/인가 웹과 관련된 공통 관심사

HTTP-> WAS -> 필터 -> 서블릿 -> 스프링 인터셉터 -> 컨트롤러

### 필터
1. 필터를 적용하면 필터 호출 후에 서블릿이 호출된다
2. 체인이 가능하다
    + 로그를 남기는 필터 적용 후, 로그인 여부 필터 적용 가능
3. 모든 요청에 대한 로깅 또는 감사
4. 웹 컨테이너에서 관리된다


### 인터셉터
1. 필터와 유사하게 인터셉터도 요청이 적절하지 않으면 자신의 상태에서 종료 가능
2. 자유롭게 체인을 추가할 수 있다
3. 스프링 컨테이너에서 관리된다

### AOP
Controller가기 바로 전에 걸치는 영역

위의 2개중에서 가장 마지막에 실행된다

### Q3
>Q3. 라이브러리 vs 프레임워크

### 프레임워크
개발자가 따라야 하는 가이드 제공
개발 할 수 있는 범위 정해짐
도구 ,플러그인 지우너

### 라이브러리 
클래스의 나열, 다른 프로그램들에서 사용 가능하도록 제공하는 방식

정리
프레임 워크는 도구이고
라이브러리는 만들어진 코드의 부분을 가져다 쓰는것 입니다.

차이점은 제어 흐름에 대한 주도성이 어디/ 누구에게 있는지에 대한 차이점이 있습니다.
